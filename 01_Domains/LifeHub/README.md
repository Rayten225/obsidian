# 🧠 Проект: LifeHub (MVP Core Specification)

## 🏗 Архитектура Ядра
1. **Навигация:** Командная строка по центру (Spotlight-style) + скрываемая визуальная док-панель (только контурные иконки, 1px).
2. **Главный Экран (Дашборд):** Настраиваемая Bento-сетка виджетов (сохраняется в локальный `dashboard.json`). Виджеты можно минимально растягивать и настраивать.
3. **Хранение текста (Knowledge Base):** Локальные чистые `.md` файлы в папках (полная совместимость с Obsidian/Notion).
4. **Хранение данных (Спорт/Задачи/Метрики):** Встроенная локальная база `SQLite`. Используется Document-Relational модель: жесткие даты, но гибкий JSON в колонке `payload` для любых цифр и массивов.
5. **Рендер (UI/UX):** Строгое разделение. Текстовый редактор отдельно, виджеты/приложения отдельно. Никаких сложных JS-блоков внутри Markdown.
6. **Управление (Command Center):** Хоткей `Ctrl+K` вызывает Spotlight. Ввод команды (например, `/timer start`) либо запускает фоновый процесс с маленьким индикатором в углу, либо разворачивает мини-виджет в полноэкранное фокус-приложение.

## 🛠 Технический Стек
* **Бэкенд:** Go (работа с файловой системой `.md`, парсинг JSON-сетки, запросы в SQLite).
* **Связка:** Wails (кроссплатформенный компилятор, легковесный бинарник).
* **Фронтенд:** Svelte (плавные встроенные анимации, компонентный подход) + CSS Grid (для Bento-сетки).
* **Визуал:** Строгий монохромный dot-matrix, глубокий черный фон, линии в 1 пиксель.

## 🚀 План разработки: Первые 4 шага в коде

### Шаг 1: Инициализация песочницы (Сегодня)
* Открыть терминал в директории проектов.
* Запустить команду генерации проекта: `wails init -n lifehub -t svelte` (или `vanilla`, если Svelte пока не ставил).
* Убедиться, что проект компилируется: запустить `wails dev` и увидеть пустое дефолтное окно.

### Шаг 2: Настройка черного холста и шрифтов
* Зайти в папку `frontend/src/`.
* Снести все стандартные стили.
* Прописать в глобальном CSS `body { background-color: #000; color: #fff; }`.
* Скачать и подключить локально открытый dot-matrix шрифт (например, из семейства JetBrains Mono или аналог NDot).

### Шаг 3: Разработка Командного Центра (Слой Svelte/JS)
* Создать компонент `CommandPalette.svelte`.
* Сверстать центрированную строку ввода, которая появляется только по нажатию `Ctrl+K` (или `Cmd+K`).
* Написать простую функцию: если в строке введено `/timer`, очистить экран и показать на весь экран огромный таймер 25:00.

### Шаг 4: Связка с Go (Сохранение лога)
* В файле `app.go` написать одну функцию: `SaveCommandLog(command string)`.
* Функция должна создавать (или дописывать) файл `inbox.md` в папке проекта, добавляя туда текст с текущим `timestamp`.
* Вызвать эту Go-функцию из Svelte при нажатии `Enter` в Командном Центре.
