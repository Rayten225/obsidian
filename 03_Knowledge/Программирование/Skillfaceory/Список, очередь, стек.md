## Список
---

***Список*** — это абстрактный тип данных представляющий собой конечную упорядоченную коллекцию данных которая предоставляет возможность обращения к любому элементу в списке, позволяет добавлять новый элемент в любое положение списка, удалять любой элемент. Каждый элемент может иметь одинаковые значения

Пример создания простейшего статического списка:
```Go
staticList := [10]int{}//Статичный, имеет макс размер - 10
```

Пример создания простейшего динамического списка:
```Go
dynList := []int{}//Динамичный, не имеет максимвльного размера
```


***Указатель*** — это тип данных, который хранит адрес. То есть в переменной типа указатель можно хранить ссылку на адрес любого участка памяти. Довольно удобный инструмент

***Односвязный список***— это список, порядок следования элементов которого связан не с физическим расположениям в памяти компьютера, а основан на подходе хранения избыточных данных, которыми являются адреса каждого следующего элемента. Благодаря этому все элементы выстраиваются друг за другом, то есть переходят в упорядоченное состояние

**Двусвязный список** — это связный список, каждый элемент которого хранит в качестве избыточных данных два указателя. Один содержит адрес предшествующего элемента (если есть), другой содержит адрес следующего элемента

**Кольцевой массив** — это тип данных, представляющий собой коллекцию элементов фиксированной длины, логически представляющую собой бесконечный набор элементов данных

## Стек
---

**Стек** — это структура данных, представляющая собой коллекцию данных (элементов), предоставляющая возможность добавления, извлечения элементов. Характерной особенностью стека является то, что добавление и извлечение элемента осуществляется с конца

## Очередь
---

**Очередь** — это структура данных, представляющая собой коллекцию элементов, для которой определены операции добавления элемента и извлечения элемента. Причём добавление нового элемента осуществляется в конец, а извлечение происходит из начала
	-  **Статическая очередь** — это очередь с фиксированным максимальным размером элементов, которые она может вместить. По достижении этого размера добавление новых элементов невозможно и должно приводить к ошибке	
	- **Динамическая очередь** — очередь изменяемого размера. Размер автоматически настраивается так, чтобы можно было поместить требуемое количество элементов

**Двусторонняя очередь** — это разновидность интерфейса очереди, позволяющая явно не задавать порядок добавления и извлечения элементов. В такой реализации вставка и извлечение возможны как с головы очереди, так и с хвоста.
- Первый метод — **queueHead()**. Вставка со стороны начала очереди
- Второй метод — **dequeueTail()**. Позволяет извлечь элемент из хвоста очереди
