## **Алгоритм** 
---
***Алгоритм*** — это набор действий приводящих к какому то результату

***Эффективность алгоритма*** — это оценка того как много ресурсов потребляет алгоритм для выполнения

Зависимость количества итерации в линейном поиске можно представить как 
**_y_(_x_) = _x_**, где _**y**_ — это количество попыток, которое нам понадобится, а _**x**_ — размер входных данных

## Структура данных
---

**Структура данных** — это контейнер, который хранит данные в определённом формате (с определённой структурой)

- **Массив** (англ. _array_) — это последовательность фиксированной длины из нуля и более элементов определённого типа

- **Список** (англ. _list_) — это последовательность элементов, где каждый элемент может быть связан с другим с помощью ссылки на следующий и/или предыдущий элемент

- **Очередь** (англ. _queue_) — абстрактный тип данных, представляющий собой множество элементов, организованных по принципу _FIFO_ (англ. _first in_ — _first out_, первым пришёл — первым вышел)

- **Стэк** (англ. _stack_) — абстрактный тип данных, представляющий собой множество элементов, организованных по принципу _LIFO_ (англ. _last in_ — _first out_, последним пришёл — первым вышел)

- **Граф** моделирует набор связей, они состоят из узлов (они же вершины) и рёбер (они же дуги). Узел представляет собой какую-то сущность, а рёбра указывают на связи. Узел может быть напрямую соединён с несколькими другими узлами. Эти узлы называются соседями

- **Дерево** — это связный ациклический граф. Связность означает наличие маршрута между любой парой вершин, ацикличность — отсутствие циклов
	- **Бинарное дерево** — это дерево, в котором у каждого узла существует не более двух потомков
	- **Бинарное дерево поиска** — это бинарное дерево, у которого значение левого потомка меньше значения родителя, а значение правого потомка больше значения родителя для каждого узла дерева
	- **Сбалансированное бинарное дерево поиска** — это бинарное дерево поиска с логарифмической высотой, поиск по которому будет иметь сложность **_O_(log(_n_))**

## **Хеш-таблицы, Хеш-мапы**
---

**Хэш функция** — это функция которая превращает любой объект в набор байтов ограниченной длины. Она всегда возвращает один и тот же результат одних и тех же входных данных

**Хэш** — это результат работы хэш-функции, те самые байты, которые она возвращает

**Коллизия** — это когда хэш-функция возвращает один и тот же хэш для разных входных данных

**Хеш-таблица** (хеш-мапа) — это структура данных, позволяющая хранить пары (ключ, значение)

Слайс — это массив который увеличивается при заполнении. У него есть два поля:
- `len` (англ. _length_ — длина) — тут содержится “виртуальная” длина массива, то есть количество ячеек, которые задействованы и доступны для использования извне.
- `cap` (англ. _capacity_ — вместимость) — реальная длина массива

Инициализировать слайс можно вот так 
```Go
ar := make([]int, 2, 4)// где 2 это длинна массива которая будет автоматом заполнена нулями и 4 - capacity
```
