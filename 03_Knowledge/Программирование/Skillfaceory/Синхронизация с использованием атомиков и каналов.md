## Условная переменная 
---
Условная переменная — это переменная которая реализовывает мьютекс и может блокировать горутину 

Пример использования
```Go
c := sync.NewCond(&sync.Mutex{}) // Создание переменной мьютекса 

c.L.Lock() // Блокировка кода для остальных горутин 
//Some code 
c.L.Unlock() // Разблокировка доступа к функции 

c.Wait()//Остановка горутины в ожидании сигнала 
c.Signal()// Сам сигнал который разблокирует последнюю заблокированную горутину 
c.Broadcast() //Сигнал который разблокирует все горутины ожидающие сигнала 
```

## Пакет Atomic
---
Этот пакет стоит использовать аккуратно так как он низкоуровневый и может произойти ситуация одновременного чтения и изменения одной ячейки памяти 

Пример использования
```Go
atomic.AddInt64(&counter, step) //counter-это адрес памяти который будет изменяться, а step-это число на которое будет увеличиваться переменная 
```

## Каналы
---
Каналы — это основной инструмент взаимодействия горутин в языке Go

Пример использования 
```Go
var c chan int //Создание неинициалтзтрованного канала 
var c chan int = make(chan int, 5) //Создание инициализированного канала с размером 5  
cRecv := make(<-chan string) //Создание канала только для извлечения данных 
cSend := make(chan<- string) //Создание канала только для записывания данных 

c <- 64 //Запись значения в канал 
fmt.Println(<-c) //Чтение из канала 

//Так же канал можно передавать и возвращать из функций 
func doAWork(c <-chan int) o <-chan int {
     // Код
}

msg, isClosed := <-c //У каналов есть еще один возвращаемый лемент 'isClosed' который возвращает true или false в зависимости от того закрыт канал или нет 

close(c) //Функуция закрытия канала
```

## Общение горутин с помощью каналов
---

Каналы можно использовать вместо уловных переменных блокируя каналы ожиданием чтения из него. При закрытии канала все заблокированные горутины которые ожидали сообщения возобновят свою работу 

Пример использования 
```Go
const goroutineAmount int = 5
 
func main() {
    begin := make(chan interface{})
    var wg sync.WaitGroup
    // Цикл запуска пяти горутин
    for i := 0; i < goroutineAmount; i++ {
        wg.Add(1)
        go func(i int) {
            defer wg.Done()
            <-begin
            fmt.Printf("Горутина №%d получила сигнал о закрытии и 
                        завершила свою работу\n", i)
        }(i)
    }
    fmt.Println("Оповещение и разблокировка горутин произойдёт через 5 
                 секунд...")
    // Некий другой продолжительный по времени выполнения код,
    // вместо которого - просто блокировка на 5 секунд
    time.Sleep(time.Second * 5)
    close(begin)
    wg.Wait()
}
```

Так же в Go есть конструкция select которая напоминает switch в других языках

Пример использования
```Go
select { 
    case <- chan1: 
    // Логика, которую следует реализовать 
    // при получении сообщения из канала chan1 
    case <- chan2: 
    // Логика, которую следует реализовать 
    // при получении сообщения из канала chan2
    case <- chan3: 
    // Логика, которую следует реализовать 
    // при получении сообщения из канала chan3
}
```


## Пакет time
---
Создать простой канал, в который через заданный период автоматически отправится сообщение в виде текущего времени, можно так:

```Go
 var c <-chan time.Time = time.After(5 * time.Second)


for {
    <-time.Tick(time.Second * 1)
    // Логика, которую следует реализовать
    // периодически с заданным шагом во времени (1 секунда)
}
```
