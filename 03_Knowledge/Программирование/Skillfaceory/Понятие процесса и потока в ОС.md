**Процессор** (_Central Processing Unit, CPU_) вызывает команды из памяти, определяет их тип и выполняет одну за другой. При необходимости передаёт результаты обратно в память или в устройства ввода-вывода.

**Шина** _(Bus)_ — канал передачи данных между частями процессора.

**Регистр** _(Hardware Register)_ — в данном случае, устройство памяти внутреннее для процессора.

**Блок управления** _(Control Unit)_ в процессоре отвечает за вызов команд из памяти и определение их типа.

**АЛУ** (арифметико-логическое устройство) — _Arithmetic logic unit (ALU)_ — выполняет операции над данными.

> **Прерывания** позволяют приостановить нормальное выполнение цикла команд, и передать управление устройствам **ввода-вывода** _(I/O-devices)_

**Многозадачность** _(multitasking)_ — способность выполнять несколько команд одновременно. Это может быть достигнуто в том числе с помощью использования прерываний (пока одна команда ждёт ответа по сети или ввода-вывода, другая команда выполняется) и усовершенствования архитектуры процессора (внедрением параллелизма на уровне команд)

> **Процесс** _(process)_ — это сущность, которая может быть назначена процессору и выполнена на нём, состоящая из программного кода и набора данных, связанных с этим кодом

Процесс характеризуется рядом элементов, включая:

- **Идентификатор** — уникальный _ID_ процесса; нужен, чтобы отличать его от всех прочих процессов.
- **Состояние** — описание текущего статуса процесса; в минимальном варианте два возможных значения: выполняется / не выполняется. 
- **Приоритет** — уровень приоритета относительно других процессов.
- **Программный счётчик** — адрес очередной выполняемой команды.
- **Указатели памяти** — хранят указатели на код и данные текущего процесса, а также на блоки памяти, используемые совместно с другими процессами.
- **Данные контекста** — данные, содержащиеся в регистрах процессора во время выполнения процесса.
- **Информация о состоянии ввода-вывода** — включает внешние запросы ввода-вывода, список устройств ввода-вывода и файлов, используемых процессом.
- **Учётная информация** — может содержать количество процессорного времени, время работы, учётные записи и прочее.

**Многопоточностью** называется способность операционной системы поддерживать в рамках одного процесса несколько параллельных путей выполнения  

**Можно выделить четыре основных типа планирования:**

|                            |                                                                                                                         |
| -------------------------- | ----------------------------------------------------------------------------------------------------------------------- |
| **Тип планирования**       | **Описание**                                                                                                            |
| Долгосрочное планирование  | Решение о добавлении процесса в пул выполняемых процессов.                                                              |
| Среднесрочное планирование | Решение о добавлении процесса к числу процессов, полностью или частично размещённых в основной памяти.                  |
| Краткосрочное планирование | Решение о том, какой из доступных процессов будет выполняться процессором.                                              |
| Планирование ввода-вывода  | Решение о том, какой из запросов процессов на операции ввода-вывода будет обработан доступным устройством ввода-вывода. |

**Базовые стратегии планирования:**

|   |   |
|---|---|
|Первым поступил — первым обслужен _(**FCFS** — first come — first served)_|Схоже с принципом очереди _(FIFO_ _—_ _first in_ _—_ _first out)_. Как только процесс становится готовым к выполнению, он присоединяется к очереди готовых процессов. Следующим выполняется процесс, который дольше всех был в очереди.|
|Круговое (карусельное) планирование _(**RR** — round robin)_|Каждый выбранный интервал времени выполняющийся процесс помещается в очередь готовых к выполнению, а следующий процесс выбирается по принципу _FCFS_.|
|Выбор самого короткого процесса _(**SPN** — shortest process next)_|На выполнение отправляются в первую очередь быстро выполняемые процессы. Сложность состоит как раз в оценке времени выполнения.|
|Наименьшее оставшееся время _(**SRT** — shortest remaining time)_|Вытесняющая версия _SPN_. Если время на выполнение нового процесса в очереди меньше оставшегося времени текущего, текущий может быть вытеснен.|
|Наивысшее отношение отклика _(**HRRN** — highest response rate next)_|Рассчитывается [![](https://latex.codecogs.com/gif.latex?R=\frac{w+s}{s} "R=\frac{w+s}{s}")](https://www.codecogs.com/eqnedit.php?latex=R=\frac{w+s}{s}), где _w_ _—_ время ожидания процессом процессора, _s_ _—_ ожидаемое время выполнения процесса. Выбирается на выполнение процесс с наивысшим параметром _R_.|
|Снижение приоритета _(**FB** — feedback)_|Перемещает долго выполняющиеся процессы в очереди с более низким приоритетом.|
