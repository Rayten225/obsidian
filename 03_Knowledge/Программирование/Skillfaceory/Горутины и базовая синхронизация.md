
## Горутины
---

***Горутина*** — это легковесный поток который работает параллельно с другими горутинами в том же адресном пространстве 

Горутина запускается с помощью `go` перед функцией 

***Семафор*** — это инструмент синхронизации у которого есть два действия увеличение и уменьшение значения на единицу, если уменьшение нулевого значения приведет к блокировке.
По большей части семафоры нужны для регулировки нагрузки к определенной части кода. Например: запросы к бд, запросы к API и тп.

***Мьютекс*** — это инструмент синхронизации обеспечивающий взаимное исключение использования критических участков кода, отличие от семафора тем что у мьютекса есть владелец который должен его освобождать 

***Спинлок*** — это низкоуровневый инструмент синхронизации который позволяет блокировать участок кода заставляя потоки постоянно проверять доступен ли этот участок кода в данный момент при этом не блокируя сам поток 
## Mutex
---

Чтобы использовать мьютекс, нужно создать объект типа `Mutex` из пакета `sync`:

```Go
mutex := sync.Mutex{}
```

Используя методы `Lock()` и `Unlock()`, вы можете заблокировать и освободить мьютекс:

```Go
mutex.Lock()
sharedValue = newCalculatedValue
mutex.Unlock()
```

## RWMutex
---
RWMutex — то же самое что и Mutex, но в нем можно блокировать отдельно чтение и запись 

У _RWMutex_ есть следующие методы:

```Go
mutex := sync.RWMutex{}

mutex.RLock()
// Блокировка только для чтения
mutex.RUnlock()

mutex.Rock()
// Блокировка для записи
mutex.Unlock()
```
